## Why this matters

The repo carries a broken, unused auth module, three unused authentication
dependencies (one of them a native module that slows installs), a second
lockfile that will silently drift from the real one, and a full duplicate toast
system. This is dead weight that confuses contributors and, in the case of
`src/lib/auth.ts`, is actively broken code that only survives because
`next.config.ts` sets `ignoreBuildErrors: true`. Removing it first gives every
later plan a clean base and removes 6 of the 24 TypeScript errors that Plan 007
must clear before it can turn typechecking back on.

## Current state

- `src/lib/auth.ts` — exports `handleAuth`. **Never imported anywhere**
  (`grep -rn "handleAuth" src/` returns only its own definition). It references
  `User` fields that do not exist in `prisma/schema.prisma` (`clerkId`, `name`,
  `image`) and uses the Pages-Router `NextApiRequest` + the Clerk v5
  `clerkClient.users` API. It produces 6 of the current TypeScript errors.
- `package.json` dependencies that are **not imported anywhere in `src/`**
  (`grep -rn "bcrypt\|next-auth\|@next-auth"` in `src/` returns nothing):
  - `"next-auth": "^4.24.10"`
  - `"@next-auth/prisma-adapter": "^1.0.7"`
  - `"bcrypt": "^5.1.1"` (native addon — no import, no usage)
- Two lockfiles exist: `package-lock.json` (npm) and `pnpm-lock.yaml` (pnpm).
  This project uses **pnpm** (the `pnpm-lock.yaml` is the source of truth). The
  `package-lock.json` is stale duplication.
- Dead toast system — the app renders `sonner`'s `<Toaster>` (`src/app/layout.tsx:8,96`)
  and every call site does `import { toast } from 'sonner'`. The shadcn toast
  trio below is **not imported by any app file** (only self-referential):
  - `src/hooks/use-toast.ts`
  - `src/components/ui/toaster.tsx`
  - `src/components/ui/toast.tsx`
- Dead imports / constants:
  - `src/app/layout.tsx:6` — `import { Footer } from '@/components/custom/footer'`
    is never rendered in the root layout.
  - `src/app/(dashboard)/dashboard/library/page.tsx:5` — `Card, CardContent`
    imported, never used. Line `11` — `revalidateLibrary` imported, never used.
  - `src/app/(dashboard)/dashboard/library/actions.ts:131` —
    `const RATE_LIMIT_DELAY = 1000` is never referenced.
  - `src/app/(dashboard)/dashboard/books/actions.ts:3` — `clerkClient` is
    imported alongside `auth` but never used (only `auth` is used).

Repo conventions: this is a pnpm project (see `pnpm-lock.yaml`, and the user's
tooling). Use `pnpm`, never `npm`, for dependency changes.

## Steps

### Step 1: Delete the broken, unused auth module

Delete `src/lib/auth.ts`.

**Verify**: `grep -rn "lib/auth\|handleAuth" src/` → no matches. (The correct
user-provisioning path is `pollUser` in `src/app/(auth)/auth/actions.ts`, which
this does not touch.)

### Step 2: Delete the dead toast system

Confirm none are imported outside themselves, then delete all three:

```
grep -rn "use-toast\|ui/toaster\|ui/toast\b" src/ --include=*.tsx --include=*.ts | grep -v "src/hooks/use-toast.ts" | grep -v "src/components/ui/toaster.tsx" | grep -v "src/components/ui/toast.tsx"
```

If that command prints **any** line, STOP — something imports them. Otherwise
delete `src/hooks/use-toast.ts`, `src/components/ui/toaster.tsx`,
`src/components/ui/toast.tsx`.

**Verify**: `grep -rn "@/hooks/use-toast\|@/components/ui/toaster\|@/components/ui/toast\"" src/` → no matches.

### Step 3: Remove unused dependencies from package.json

Remove these three lines from the `dependencies` block of `package.json`:
`"next-auth"`, `"@next-auth/prisma-adapter"`, `"bcrypt"`. Then run
`pnpm install` so the lockfile updates.

**Verify**:
- `grep -n "next-auth\|@next-auth\|\"bcrypt\"" package.json` → no matches.
- `pnpm install` → exit 0.

### Step 4: Delete the duplicate npm lockfile

Delete `package-lock.json` (this is a pnpm project; `pnpm-lock.yaml` is
authoritative).

**Verify**: `ls package-lock.json` → "No such file or directory".

### Step 5: Remove dead imports and the dead constant

- `src/app/layout.tsx`: remove line `import { Footer } from '@/components/custom/footer'`.
- `src/app/(dashboard)/dashboard/library/page.tsx`: remove the unused
  `Card, CardContent` import (line ~5) and the `revalidateLibrary` name from the
  import on line ~11 (keep the other names imported from that module:
  `getBookTransaction`, `getUserLibrary`, `toggleBookFavorite`, `LibraryBook`).
- `src/app/(dashboard)/dashboard/library/actions.ts`: remove the line
  `const RATE_LIMIT_DELAY = 1000`.
- `src/app/(dashboard)/dashboard/books/actions.ts`: on line 3 change
  `import { clerkClient, auth } from '@clerk/nextjs/server'` to
  `import { auth } from '@clerk/nextjs/server'`.

**Verify**:
- `grep -n "custom/footer" src/app/layout.tsx` → no match.
- `grep -n "RATE_LIMIT_DELAY" src/` → no match.
- `grep -n "clerkClient" "src/app/(dashboard)/dashboard/books/actions.ts"` → no match.

### Step 6: Confirm the type-error count dropped by the auth.ts errors

Run `npx tsc --noEmit 2>&1 | grep -c "error TS"`.

**Verify**: the count is **18** (down from 24 — the 6 `src/lib/auth.ts` errors
are gone). If it is higher than 18, you introduced a new error — inspect the
`tsc` output and fix your own edit. If it is lower than 18, you may have deleted
something still in use — STOP and report.
